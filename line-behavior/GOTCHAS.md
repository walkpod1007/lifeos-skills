# GOTCHAS.md — line-behavior

> 格式：每條 = **錯誤描述** / **正確做法** / **觸發情境**

---

## G1: Reply API 只能送一則

**錯誤**：在同一個 reply token 送出兩則訊息，以為 5 則以內都免費沒問題。
**正確**：reply token 用完第一則就失效，第二則使用者看不到。一次只回一則，那一則必須是最完整、最優先的資訊。Flex / 圖片卡也算一則。
**觸發情境**：需要分段回覆（先送確認、再送結果）的場景。

---

## G2: Push API 額度有限，不能隨意使用

**錯誤**：把 Push API 當作一般回覆管道，沒有節制地使用。
**正確**：LINE 免費版 Push 限 200 則/月。能用 Reply 就絕不用 Push，Push 只用於無 reply token 的主動推播場景（如定時通知、任務完成）。
**觸發情境**：工具跑完後想通知使用者、heartbeat 結果推播。

---

## G3: Reply-First 原則 — 不串多步驟才回覆

**錯誤**：先做完所有工具呼叫，最後才送 Reply。若工具跑超過 60 秒，reply token 過期，使用者完全收不到。
**正確**：一次 tool call 就回覆。需要多步驟的先回目前結果（或 ACK），剩下的背景繼續做，做完用 Push 補充。目標：每則 10 秒內送出。
**觸發情境**：需要呼叫外部 API、執行長時間腳本後才有結果的任務。

---

## G4: 群組內未被 @ 就靜默

**錯誤**：在群組對話中主動插嘴，或對所有訊息都有反應。
**正確**：群組完全被動。只有被明確 @mention 才回應。系統通知/工作完成推播例外，直接 push 不需被 @。
**觸發情境**：多人群組對話，其他人在聊與 agent 無關的事情。

---

## G5: LINE heartbeat target 不能設 "line"

**錯誤**：heartbeat 設 `target: "line"` 或 `target: "last"`，導致 400 錯誤。
**正確**：LINE 是 plugin channel，不在 OpenClaw 內建 CHANNEL_IDS 列表。heartbeat 設 `target: "none"`（只跑不送）。要送通知到 LINE，改用 cron job + sessions_send。
**觸發情境**：設定 heartbeat 自動推播到 LINE 群組。

---

## G6: media 訊息需 3 秒 ACK

**錯誤**：收到圖片/語音/影片後立刻跑完整處理流程才回覆，使用者等很久以為沒收到。
**正確**：收到媒體訊息後先在 3 秒內送出 ACK（「收到，處理中…」），再背景處理，完成後用 Push 推結果。
**觸發情境**：使用者傳送圖片、語音訊息、影片檔案。
